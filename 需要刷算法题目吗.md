# 需要刷算法题目吗？

**不需要专门刷算法题目**

通过分析项目代码,这个项目的技术复杂度主要在工程设计和系统集成上,而非算法实现:

## 实际代码复杂度分析

1. **edit.ts** 是算法最复杂的地方,但核心算法只有:
   - Levenshtein 编辑距离 (166-182行) - 这是一个经典的简单字符串相似度算法
   - 多种文本匹配策略 - 都是基础的字符串操作 (split、indexOf、trim等)

2. **其他工具实现**:
   - `glob.ts`: 使用 ripgrep 库进行文件匹配
   - `bash.ts`: 使用 tree-sitter 解析 bash 命令
   - 大部分代码都是调用库 API 和业务逻辑处理

## 你需要关注的技能

1. **工程能力** > 算法能力
   - 系统设计 (Session/Agent架构)
   - TypeScript 类型系统
   - 异步编程、事件处理
   - 文件系统操作
   - CLI/TUI 交互开发

2. **基础编程能力**:
   - 基本数据结构的使用
   - 字符串处理
   - 正则表达式

3. **库和框架掌握**:
   - yargs (CLI框架)
   - zod (验证)
   - bun/node API
   - 各种 SDK (AI、LSP、tree-sitter等)

## 总结

这个项目更像是一个**工程实践项目**,而不是算法挑战题。如果你想做出类似项目,应该:

- 多做实战项目练习
- 学习系统架构设计
- 掌握常用库和工具
- 提升代码组织能力

刷算法题对你实现这个项目**帮助非常有限**,除非你的目标是进入重视算法的大厂面试。
